# 发现页动态功能设计方案

## 1. 概述

在现有「发现 → 动态」页面新增朋友圈风格动态功能，所有用户可见全局动态流，发布入口放在「我的」页面。

## 2. 产品形态

| 页面 | 路径 | 说明 |
|------|------|------|
| 全局动态流 | 发现 → 动态 Tab | 所有用户发布的动态，最新在上 |
| 我的动态 | 我的页面 → 我的动态 | 仅查看自己发布的动态 |
| 发布页面 | 我的页面 → 发布按钮 | 发布新动态（图文/视频） |

## 3. 后端设计

### 3.1 数据库

**moments 表**
```sql
CREATE TABLE moments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    content VARCHAR(500) COMMENT '文本内容，500字以内',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_created_at (created_at)
);
```

**moment_media 表**
```sql
CREATE TABLE moment_media (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    moment_id BIGINT NOT NULL,
    url VARCHAR(500) NOT NULL COMMENT '媒体URL',
    media_type TINYINT NOT NULL COMMENT '1=图片, 2=视频',
    sort_order TINYINT NOT NULL DEFAULT 0 COMMENT '排序序号',
    cover_url VARCHAR(500) COMMENT '视频封面URL',
    duration INT COMMENT '视频时长（秒）',
    FOREIGN KEY (moment_id) REFERENCES moments(id) ON DELETE CASCADE,
    INDEX idx_moment_id (moment_id)
);
```

### 3.2 API 接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 发布动态 | POST | `app/moment/create` | 文本+媒体ID列表 |
| 动态列表 | GET | `app/moment/feed` | 全局流，分页 |
| 我的动态 | GET | `app/moment/mine` | 我的动态，分页 |
| 删除动态 | DELETE | `app/moment/{id}` | 删除自己的动态 |
| 媒体上传 | POST | `app/moment/upload` | 图片/视频上传 |

**上传接口 `POST /api/v1/app/moment/upload`**
- 请求：`multipart/form-data`，字段 `file`（文件） + `type`（image/video）
- 响应：`{"url": "https://...", "media_type": 1}`
- 初期：单文件简单上传，无分片，无压缩
- 图片限制：最大 10MB，支持 jpg/png/gif/webp
- 视频限制：最大 100MB，支持 mp4/mov，最大 30 秒（前端校验）

**发布接口 `POST /api/v1/app/moment/create`**
- 请求：`{"content": "文字内容", "media_ids": [1, 2]}`（media_ids 为已上传媒体ID列表）
- 限制：图片最多 4 张，视频最多 1 个（二选一）

**动态列表 `GET /api/v1/app/moment/feed`**
- 参数：`page`（默认1）, `page_size`（默认20）
- 响应：动态列表，每条含用户信息 + 媒体列表 + 发布时间

**我的动态 `GET /api/v1/app/moment/mine`**
- 参数同上，仅返回当前用户动态

### 3.3 文件存储

初期：将文件保存到服务器本地 `uploads/moments/` 目录
- 后续可扩展：替换为 OSS/CDN，迁移时旧路径兼容

## 4. 前端设计（Flutter）

### 4.1 页面结构

```
discover_page.dart
├── _FeedTab（动态列表页）
│   └── moment_list_view.dart（可复用）
│       └── moment_card.dart（单条动态卡片）
│           └── moment_media_grid.dart（九宫格图片）
│               └── moment_video_thumbnail.dart（视频封面）

profile_page.dart（我的页面）
├── 我的动态入口（跳转 my_moments_page.dart）
└── 发布按钮（FloatActionButton，跳转 publish_moment_page.dart）

my_moments_page.dart（我的动态列表）
├── 仅自己发布的动态
└── 复用 moment_list_view.dart

publish_moment_page.dart（发布页）
├── 文本输入框
├── 图片选择（最多4张）→ 调用上传接口
├── 视频选择（最多1个）→ 调用上传接口
├── 发布按钮 → 调用创建接口
└── 预览九宫格效果
```

### 4.2 九宫格布局规则（微信朋友圈风格）

```dart
// 根据图片数量决定布局
1张图 → 宽度撑满，高度自适应（最大高度 300）
2张图 → 左右等宽，高度与宽度相同
3张图 → 第一行1张大图，第二行2张小图
4张图 → 2×2 网格
```

### 4.3 关键组件

**MomentCard（动态卡片）**
- 左侧：用户头像 + 用户名 + 发布时间
- 右侧：文字内容 + 媒体网格
- 媒体网格支持图片九宫格 + 视频封面

**MomentVideoPlayer（视频播放器）**
- 显示视频封面（首帧）
- 封面叠加播放按钮图标
- 点击后全屏/浮层播放

### 4.4 状态管理

```dart
// 新增 provider
final momentFeedProvider = FutureProvider.family<PagedList<Moment>, int>((ref, page));
final myMomentsProvider = FutureProvider.family<PagedList<Moment>, int>((ref, page));
final momentUploadProvider = StateNotifierProvider<MomentUploadNotifier, UploadState>;
```

## 5. 实现顺序

1. **后端**：数据库表 → 媒体上传接口 → 动态 CRUD 接口
2. **Flutter**：API 层（endpoints + service）→ 列表页（复用组件）→ 发布页 → 我的动态页
3. **入口**：profile_page 新增「我的动态」入口 + FloatActionButton
