# 来电可靠性改造任务（2026-04-16）

## P0（当前迭代）
- [ ] Flutter 接入 `flutter_local_notifications`，完成本地通知初始化与权限申请（Android 13+ 通知权限、iOS 通知权限）。
- [ ] 后台检测到来电时发送系统通知（标题：视频来电，正文包含主叫昵称）。
- [ ] 点击系统通知后唤起 App，并路由到来电处理流程（弹接听/拒绝或直接进入待接听页）。
- [ ] 接听/拒绝/超时后清理对应通知，避免脏通知残留。
- [ ] 通知 ID 与 `call_id` 绑定，避免多路来电互相覆盖或误清理。

## P1（下一迭代）
- [ ] 接入 FCM（Android）与 APNs（iOS）远程推送通道，后端保存设备 token（按用户维度）。
- [ ] 后端在 `dialing` 时下发来电推送事件（主叫、被叫、call_id、时间戳、幂等键）。
- [ ] 客户端收到推送后落地统一来电处理：前台弹窗、后台系统通知、回前台可继续处理。
- [ ] 推送去重策略：同一 `call_id` 仅处理一次，重复消息直接丢弃。
- [ ] 加入推送送达与处理埋点（发送成功率、通知点击率、接听率、超时率）。

## P2（体验完善）
- [ ] Android 实现标准来电样式（高优先级通知/全屏意图，按系统版本兼容处理）。
- [ ] iOS 评估并接入 CallKit + VoIP Push（如上架策略允许），统一系统来电体验。
- [ ] 来电铃声策略可配置（静音模式、震动开关、勿扰兜底文案）。
- [ ] 弱网/断网场景降级：通知可见但接听前先做网络探测与错误提示。

## 回归与验收
- [ ] 场景回归：前台来电、后台来电、App 切前台后处理来电、重复来电、主叫取消、被叫拒绝、超时自动结束。
- [ ] 双端真机验证（Android + iOS），确认通知权限拒绝时的降级提示可用。
- [ ] 与现有 RTC 信令联测，确保不影响当前“无心跳计费”的视频通话链路。
